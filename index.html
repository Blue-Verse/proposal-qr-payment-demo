프로젝트 소개

정부 정책자금 확보 및 투자 유치를 위한 QR 결제 데모 앱을 Flutter + Firebase 기반으로 신속하게 개발합니다. 실제 결제 없이 핵심 결제 흐름을 시각적으로 시연하는 데 최적화된 솔루션을 제안드립니다.

핵심 요구사항 분석

  • 사용자 앱: 간편 로그인, 가상 충전 머니 잔액 표시, 가상 충전, QR 코드 스캔 → 결제 금액 입력 → 비밀번호 입력 → 결제 완료 모의 화면, 거래 내역 조회
  • 가맹점 기능: 가맹점 고유 QR 코드 생성/표시, Mock 결제 승인 알림, 일/주/월별 매출 내역 조회
  • 관리자 페이지: 사용자 계정 생성/조회/관리, 가맹점 등록 및 QR 코드 관리, 전체 거래 내역 조회/검색
  • 중국어 UI: 모든 UI 텍스트를 중국어로 개발 (중국 관광객 대상)
  • 산출물: Android APK 파일, 전체 소스 코드, 관리자 페이지 접속 정보

제안 포인트

본 프로젝트는 시연 목적의 데모 앱으로, 실제 결제 연동 없이 핵심 결제 흐름의 시각적 완성도에 집중합니다. 다음과 같은 접근 방식으로 효율적으로 진행하겠습니다:

  • Flutter 단일 코드베이스: 사용자 앱과 가맹점 앱을 하나의 Flutter 프로젝트에서 역할 기반으로 분리 — 개발 효율 극대화
  • Firebase 서버리스 백엔드: Authentication, Firestore, Cloud Functions를 활용하여 별도 서버 구축 없이 빠르게 백엔드 구현
  • QR 코드 스캔/생성 경험: Calendar Share 프로젝트에서 QR 코드 생성 및 스캐너 기능을 구현한 경험이 있어 안정적으로 적용 가능
  • 다국어 앱 경험: DayStarter 앱에서 6개 국어(한/영/일/스페인/베트남/중국어) 스토어 리스팅 및 다국어 앱 구조를 설계한 경험 보유 — 중국어 UI 적용에 활용 가능
  • 관리자 페이지: Firebase 기반 웹 관리자 페이지를 React/Next.js로 구현하여 직관적인 관리 환경 제공

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

정부 정책자금 신청 및 투자 유치 발표를 목적으로, 중국 관광객 대상 QR 결제 서비스의 핵심 프로세스를 시각적으로 시연하기 위한 데모 앱을 개발합니다. 실제 결제 시스템이나 금융 보안 인증 없이, 핵심 기능(QR 스캔 → 결제 모의)에 집중한 시연용 앱입니다.

2. 사용자 정의

사용자 유형설명주요 기능
일반 사용자 (소비자)중국 관광객 역할의 데모 사용자간편 로그인, 잔액 조회, 가상 충전, QR 결제, 거래 내역
가맹점주QR 결제를 수신하는 상점 역할QR 코드 생성, 결제 승인 알림 수신, 매출 내역 조회
관리자시스템 전체 관리자사용자/가맹점 관리, 거래 내역 관리

3. 기능 요구사항

3.1 사용자 앱 — 소비자

  • [필수] 간편 로그인 — 데모용 사전 등록 계정으로 로그인
  • [필수] 잔액 조회 — 가상 충전 머니 잔액 실시간 표시
  • [필수] 가상 충전 — 버튼 클릭 시 정해진 금액 충전 (100위안, 500위안, 1000위안)
  • [필수] QR 결제 시뮬레이션:
    • 앱 내 카메라를 통한 QR 코드 스캔
    • 가맹점 정보 확인 및 결제 금액 입력
    • 결제 비밀번호 입력 (6자리 PIN)
    • 결제 완료 모의 화면 (성공 애니메이션)
  • [필수] 거래 내역 — 충전 및 결제 내역 목록 (날짜, 금액, 가맹점명)

3.2 사용자 앱 — 가맹점주

  • [필수] QR 코드 생성 — 가맹점 고유 QR 코드 생성 및 화면 표시
  • [필수] 결제 승인 알림 — 소비자 결제 시 가상 결제 승인 알림 (푸시 또는 인앱)
  • [필수] 매출 내역 조회 — 일/주/월별 가상 매출 통계 및 내역

3.3 관리자 페이지 (웹)

  • [필수] 사용자 관리 — 데모용 사용자 계정 CRUD
  • [필수] 가맹점 관리 — 가맹점 등록, QR 코드 정보 관리
  • [필수] 거래 내역 관리 — 전체 거래 내역 조회/검색/필터링

4. 비기능 요구사항

항목요구사항
언어모든 UI 텍스트 중국어 (간체)
플랫폼Android (APK 파일 배포)
디자인시연에 적합한 깔끔하고 직관적인 UI
성능데모 시연 시 지연 없는 반응 속도
데이터Firebase Firestore 기반 실시간 동기화

5. 기술 제약사항

  • Flutter 또는 React Native 기반 (Flutter 권장)
  • Firebase 백엔드 (Authentication, Firestore, Cloud Functions)
  • 관리자 페이지: React/Next.js 기반 웹 애플리케이션
  • QR 코드: qr_flutter (생성) + mobile_scanner (스캔)

6. 범위 외 사항 (Out of Scope)

  • 실제 결제 시스템 연동 (알리페이, 위챗페이 등)
  • 금융 보안 인증 (PCI-DSS 등)
  • iOS 앱 개발 및 App Store 배포
  • Google Play Store 배포 (APK 직접 설치)
  • 중국어 번역 (별도 협의 시 추가 가능)
  • 실시간 환율 연동

Calendar Share (소셜 캘린더 앱)

소셜 네트워킹 / 일정 관리 · 2025

프로젝트 개요

소셜 네트워킹과 캘린더 일정 관리를 통합한 크로스플랫폼 모바일 앱. Flutter 단일 코드베이스로 Android/iOS 동시 지원, Firebase 서버리스 백엔드 활용.

핵심 기능

  • 소셜 캘린더: 포스트+이벤트 융합 모델, 45+ 화면
  • QR 코드 생성 및 스캐너 — 오프라인 소셜 디스커버리
  • Firebase 서버리스 (Auth, RTDB, Storage, FCM)
  • 실시간 메시징, 7종 푸시 알림

본 프로젝트와의 유사점

Flutter + Firebase + QR 코드라는 동일한 기술 조합. QR 코드 생성 및 스캔 기능을 실제 구현한 경험이 있어 본 프로젝트의 QR 결제 스캔 플로우에 직접 활용 가능합니다. Firebase 서버리스 아키텍처 설계 노하우도 그대로 적용됩니다.

본 프로젝트와의 유사점

Flutter + Firebase 기반 3개월 빠른 딜리버리 경험. 기획부터 디자인, 개발, 출시까지 전 과정을 혼자 수행한 경험이 있어 본 프로젝트도 효율적으로 진행할 수 있습니다. Clean Architecture 패턴을 적용하여 코드 품질을 보장합니다.

헬스케어 / 게이미피케이션 · 2022.06~ (3년 운영)

프로젝트 개요

센서 기반 수면 인증 + 게이미피케이션 알람 앱. Flutter 기반으로 기획, 디자인, 개발 전 과정을 수행하며 3년 이상 실서비스 운영 중.

핵심 기능

  • Spine 2D 캐릭터 시스템, 7단계 루트박스 이코노미
